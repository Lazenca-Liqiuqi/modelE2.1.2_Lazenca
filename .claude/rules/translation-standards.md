# ModelDescription文档翻译规范

## 一、文件位置规范

### 1.1 目录结构原则

翻译文件必须保持原有目录结构，不得创建新的子目录。

**规则**：
- 原始文件：`old-doc/ModelDescription/*.html`
- 翻译文件：`doc/ModelDescription/*.md`

**反例**：
```
❌ doc/technical-reference/atmosphere/
❌ doc/technical-reference/land/
```

**正例**：
```
✅ doc/ModelDescription/
```

### 1.2 文件命名

- 保持原文件名，仅改变扩展名：`.html` → `.md`
- 例如：`Atmospheric_model.html` → `Atmospheric_model.md`

## 二、术语翻译规范

### 2.1 避免口语化表达

技术术语必须使用标准、正式的中文表达，避免口语化或过于简略的译法。

| 不推荐译法 | 推荐译法 | 说明 |
|-----------|----------|------|
| 植被土 | 植被覆盖土壤 | "植被土"过于口语化 |
| 变深度层 | 可变深度层 | 加上"可"字更规范 |
| 根深 | 根系深度 | 避免过于简略 |
| 地表蓄水层 | 地表蓄水 | "层"字可能引起歧义 |

### 2.2 专有名词处理

**原则**：保留英文专名，必要时用中文说明。

| 英文术语 | 处理方式 | 示例 |
|---------|----------|------|
| TOPmodel | 保留英文 | "TOPmodel方法" |
| NASA GISS | 保留英文 | "NASA GISS ModelE" |
| MathJax | 保留英文 | "MathJax系统" |

### 2.3 新术语处理流程

遇到词典中未收录的术语时：
1. 查阅相关领域权威资料
2. 选择最标准、最通用的译法
3. 记录到 `.claude/terminology-dictionary.md` 供后续参考

## 三、句式翻译规范

### 3.1 避免"直译+括号解释"模式

**反例**：
```
蒸散发从地表蓄水层和下部（作为根深函数的）获取水分。
```

**正例**：
```
蒸散发从地表蓄水与下部土层取水，其取水深度是根系深度的函数。
```

**改进原则**：
- 将括号内的解释融入主句
- 使用"是...的函数"等自然中文表达
- 避免生硬的"作为...的"结构

### 3.2 保持句子流畅性

**检查方法**：
- 朗读译文，检查是否通顺
- 避免欧化句式
- 确保技术术语与日常用语搭配自然

**示例**：
```
❌ 可具有可变范围
✅ 其覆盖范围可变
```

## 四、格式规范

### 4.1 中英对照叠放格式

**标准格式**：
```markdown
# Title / 标题

English paragraph one.
英文段落一。

English paragraph two.
英文段落二。
```

**要求**：
- 标题：`# English / 中文`
- 段落：英文在上，中文在下
- 中英文之间空一行
- 保持原有段落结构

**段落拆分规则**：
- 允许将源HTML的单个`<p>`段落按句拆分为多组句对，以提升可读性
- 拆分时保持句子顺序不变，按句号、问号等自然断点拆分
- 每组句对保持"英文+中文"的叠放格式

### 4.2 特殊内容处理

#### LaTeX公式
- 公式本身不翻译
- 说明文字需要翻译

**示例**：
```markdown
Here is an example of in-line formula: $W_i = x$
这是一个行内公式示例：$W_i = x$
```

#### 文献引用
- 保持原格式不变
- 作者姓名可音译

**示例**：
```
(Stieglitz, 1994)
(Stieglitz, 1994)
```

#### 代码块
- 代码内容不翻译
- 注释可翻译

## 六、常见问题与解决方案

### 问题1：术语词典未覆盖
**解决方案**：
1. 查阅专业文献
2. 选择标准译法
3. 更新词典文件

### 问题2：原文拼写错误
**示例**：`Evapotransiration` → `Evapotranspiration`，`intereaction` → `interaction`
**处理方式**（已统一策略）：
- 在译文中更正为正确拼写
- 在中文译文后添加括号说明"（原文拼写：xxx）"以保留可追溯性
- 这样既保证了正确性，又保留了与原文的对应关系

### 问题3：测试内容是否翻译
**原则**：
- 测试说明文字：翻译
- 测试代码/公式：不翻译
- 标题：翻译

## 八、经验教训总结（基于任务#12/#18/#20审查）

### 8.1 术语词典一致性
**问题**：restart file译为"重启动文件"，与词典v1.4的"重启文件"不一致
**教训**：必须始终与术语词典保持一致，词典是唯一的术语标准来源
**修正**：
- 翻译前必须查阅术语词典v1.4
- 如词典译法不妥，先更新词典，再统一使用
- 不得自行采用其他译法，哪怕看起来更合理

### 8.2 语义误导风险
**问题**："deposited"译为"沉积"，在地质学中易被理解为"沉积物沉降"
**教训**：选择译词时必须考虑专业领域的常见含义，避免歧义
**修正**：
- "deposited"在水文学语境下译为"分配/加入/注入"
- 检查译词在目标领域是否有其他常见含义
- 优先选择含义单一、明确的表达

### 8.3 重复结构优化
**问题**："超过预定义门槛深度的深度"出现重复的"深度"
**教训**：避免在短距离内重复相同词汇，影响可读性
**修正**：
- "超过预定义门槛深度的那部分水深"
- 使用代词或量词替代重复词汇
- 保持句式清晰流畅

### 8.4 原文拼写错误处理策略
**问题**：原文intereaction、Tranpsort拼写错误，直接更正后丢失可追溯性
**教训**：必须在更正错误的同时保留原文信息
**修正**（已统一策略）：
- 在译文中使用正确拼写
- 在中文译文后添加"（原文拼写：xxx）"说明
- 确保读者可以回溯到原文

### 8.5 段落结构保真口径明确
**问题**：源HTML单段<p>被拆分为多组句对，与"保持原有段落结构"规范存在冲突
**教训**：必须明确"段落结构保真"的具体含义，避免模糊地带
**修正**（已在4.1节中明确）：
- 允许按句拆分以提升可读性
- 拆分时保持句子顺序不变
- 每组句对保持"英文+中文"叠放格式

### 8.6 新术语及时补录
**问题**：本批次多个新术语未在词典v1.4中，可能导致后续译名分裂
**教训**：每批次翻译完成后，必须及时补录新术语到词典
**需要补录的术语**：
- driver routine → 驱动例程
- prognostic variables → 预报变量
- runoff → 径流
- downstream direction → 下游方向
- sill depth → 门槛深度
- heat diffusion → 热扩散
- convective overturning → 对流翻转
- temperature stratification → 温度层结

## 十一、经验教训总结（基于任务#11/#9/#8审查）

### 11.1 同根术语一致性

**问题**：Vegetation_model.md中conductance相关术语译法不一致
- stomatal conductance → 气孔导度 ✅
- water vapor conductance → 水汽传导 ❌
- canopy conductance → 冠层传导 ❌

**教训**：同根术语必须保持译词一致，避免同根不同译
**修正**：
- 统一为"导度"：水汽导度、冠层导度、植被导度
- 翻译时检查同一词根的所有相关术语

### 11.2 术语辨识度不足

**问题**：SUBSIDENCE译为"下沉循环"，与DOWNDRAFT"下沉气流"中文高度同形，术语辨识度不足
- SUBSIDENCE = 补偿环境下沉（质量守恒补偿）
- DOWNDRAFT = 对流尺度下沉气流（负浮力驱动）

**教训**：不同物理过程的术语需要足够区分度，避免中文同形
**修正**：
- SUBSIDENCE → 补偿环境下沉/环境下沉循环
- 在解释中明确其"补偿上升/下沉质量通量"的背景

### 11.3 位置概念错误

**问题**：EVAP_PRECIP段首句将cloud top误译为"云底"
- 原文：`loops from the level below cloud top (LMAX-1)`
- 误译：从**云底**以下的层级（`LMAX-1`）…

**教训**：位置概念错误属于P0级严重问题，必须准确理解cloud top/base等空间关系
**修正**：仔细核对云顶（cloud top）和云底（cloud base）的所有相关表述

### 11.4 口语化表达需避免

**问题**："根深"过于口语化，不够规范
- 不推荐：根深、根深分布
- 推荐：根系深度、根系深度分布

**教训**：选择技术术语时需考虑规范性和专业性，避免过于简略
**修正**：参考translation-standards.md:3.2示例，使用"根系深度"

### 11.5 新术语及时补录

**问题**：第三批文档涉及大量新术语未在词典v1.4中
- 植被模型术语：canopy、biogeochemistry、phenology、LAI等
- PBL术语：roughness length、drag coefficient、Monin-Obukhov length等
- 云微物理术语：entrainment、detrainment、autoconversion、accretion、glaciation等
- 湍流术语：second-order closure、nonlocal transport等

**教训**：每批次翻译完成后，必须及时补录新术语到词典，避免后续译名分裂
**需要补录的术语**（17个）：
- canopy conductance / water vapor conductance → 冠层/水汽导度
- root depth distribution → 根系深度分布
- entrainment / detrainment → 卷入 / 夹卷
- autoconversion / accretion → 自动转化 / 碰并
- glaciation / Bergeron-Findeisen process → 冰川化 / Bergeron-Findeisen过程
- Monin-Obukhov length → Monin-Obukhov长度
- roughness length → 粗糙长度
- drag coefficient → 曳力系数
- Stanton number / Dalton number → Stanton数 / Dalton数
- nonlocal vertical transport → 非局地垂直输送
- second-order closure (SOC) → 二阶闭合（SOC）
- bulk Richardson number → 总体理查森数

### 11.6 原文拼写错误处理一致性

**问题**：本批次多处原文拼写错误需要统一处理策略
- Farqhuar → Farquhar（Vegetation_model.html）
